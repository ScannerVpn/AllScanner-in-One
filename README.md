# 🛡️ VPN Scanner Suite

هشت اسکنر VPN در یک برنامه‌ی واحد با تب‌بندی بالای صفحه. با یک بار اجرا، همه‌ی اسکنرها بالا می‌آیند و می‌توانی بین آن‌ها جابجا شوی:

- **Surfshark**
- **NordVPN**
- **ExpressVPN**
- **PureVPN**
- **Mullvad**
- **PIA** (Private Internet Access)
- **Windscribe**
- **Proton VPN**

## اجرا

### روش ساده (ویندوز)
فایل `Start.bat` را دابل‌کلیک کن. وابستگی‌های لازم نصب می‌شوند، همه‌ی اسکنرها بالا می‌آیند و مرورگر به‌صورت خودکار باز می‌شود.

### ساخت نسخه‌ی نصبی
برای ساخت نسخه‌ی نصبی ویندوز از `build-installer.bat` استفاده کن:

```bat
build-installer.bat
```

این اسکریپت به ترتیب:
- وابستگی‌های ریشه‌ای را نصب می‌کند
- Node باندل‌شده را دانلود می‌کند
- نصب‌کننده‌ی ویندوز را با `electron-builder` می‌سازد

نتیجه در پوشه‌ی `dist\` قرار می‌گیرد.

### روش دستی
```bash
git clone https://github.com/ScannerVpn/AllScanner-in-One.git
```
```bash
cd AllScanner-in-One
```
```bash
npm install
```
```bash
npm start
```
سپس مرورگر روی `http://localhost:8080/` باز می‌شود.

## 🖥️ اپ دسکتاپ (ویندوز)

نسخه‌ی دسکتاپ با **Electron** ساخته می‌شود؛ کاربر نهایی هیچ‌چیز (Git/Node/npm) نصب نمی‌کند و فقط نصب‌کننده را اجرا می‌کند.

```bash
npm install            # وابستگی‌ها + دانلود خودکار Node باندل‌شده
npm run app            # اجرای اپ در حالت توسعه
npm run dist           # ساخت نصب‌کننده → dist/VPN Scanner Suite Setup *.exe
```

ویژگی‌های نسخه‌ی دسکتاپ:
- **پورت داینامیک:** هر اسکنر روی یک پورت آزاد بالا می‌آید، پس تداخل پورت رخ نمی‌دهد. قفل تک‌نمونه‌ای هم جلوی اجرای دوباره را می‌گیرد.
- **بدون پاپ‌آپ فایروال:** همه‌ی اسکنرها روی `127.0.0.1` گوش می‌دهند.
- **Node باندل‌شده (OpenSSL):** اسکنرها با یک Node واقعی (نه Node داخل Electron) اجرا می‌شوند. دلیلش مهم است: Node داخل Electron از **BoringSSL** استفاده می‌کند و اثرانگشتِ TLS ClientHello متفاوتش پشت DPI ریست می‌شود و همه‌ی سرورها «قرمز» می‌شوند؛ با Node مبتنی بر **OpenSSL** رفتار دقیقاً مثل حالت وب است. این باینری در گیت نیست و با `npm run setup-node` (یا خودکار هنگام `npm install`/`npm run dist`) از nodejs.org دانلود می‌شود.

> توجه: اکزه امضای دیجیتال ندارد، پس ویندوز بار اول اخطار SmartScreen «ناشر ناشناس» می‌دهد → **More info → Run anyway**.

## ساختار

```
vpn-scanner-suite/
├── suite.js          # نقطه‌ی ورود واحد: هر ۴ اسکنر را بالا می‌آورد + سرور پوسته
├── shell.html        # صفحه‌ی تب‌دار که هر اسکنر را در iframe نشان می‌دهد
├── package.json
├── Start.bat         # اجرای یک‌کلیکی ویندوز
└── scanners/
    ├── surfshark/    # پورت 3002 (Node خام)
    ├── nord/         # پورت 3000 (Node خام)
    ├── expressvpn/   # پورت 3003 (Express)
    ├── purevpn/      # پورت 3004 (Express)
    ├── mullvad/      # پورت 3005 (Express)
    ├── pia/          # پورت 3006 (Express)
    ├── windscribe/   # پورت 3007 (Express)
    └── proton/       # پورت 3008 (Express)
```

## چطور کار می‌کند

- `suite.js` هر اسکنر را به‌صورت یک پروسه‌ی جداگانه روی پورت خودش اجرا می‌کند (کد اسکنرها دست‌نخورده می‌ماند).
- یک سرور سبک روی پورت **8080** صفحه‌ی `shell.html` را سرو می‌کند که شامل نوار تب و یک `iframe` برای هر اسکنر است.
- هر iframe داشبورد اصلی همان اسکنر را از پورت خودش بارگذاری می‌کند؛ بنابراین تمام قابلیت‌های هر اسکنر بدون تغییر کار می‌کنند.
- **بارگذاری تنبل:** هر اسکنر فقط وقتی اولین بار تبش را باز کنی لود می‌شود.

## پورت‌ها

| اسکنر | پورت |
|--------|------|
| پوسته (تب‌ها) | 8080 |
| NordVPN | 3000 |
| Surfshark | 3002 |
| ExpressVPN | 3003 |
| PureVPN | 3004 |
| Mullvad | 3005 |
| PIA | 3006 |
| Windscribe | 3007 |
| Proton VPN | 3008 |

اگر پورتی اشغال بود، می‌توانی پورت پوسته را تغییر بدهی:
```bash
SHELL_PORT=9000 npm start
```

## نکته‌ی مهم هنگام اسکن
برای نتیجه‌ی درست، قبل از اسکن، اپ VPN را کاملاً **خاموش** کن؛ وگرنه ترافیک از داخل تونل عبور می‌کند و همه‌ی سرورها اشتباهاً «در دسترس» نشان داده می‌شوند.
